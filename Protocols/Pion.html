Pion

public protocol Pion

Ce protocol représente un pion du jeu. Un pion appartient à un joueur, se situe à une certaine position sur le plateau et est en vie ou pas.

  • init(estMaitre:joueur:position:)

    Crée un pion.

    Important

    x appartient à [0 ; 4] et y appartient à [0 ; 4], si ce n'est pas le cas, la méthode émet une erreur fatale

    Suite à l'appel init(estMaitre: estMaitre, joueur: joueur, position: position) on a :

    self.estMaitre == estMaitre
self.joueur.estJoueur1() == joueur.estJoueur1()
self.position == position
self.enVie == true

    Declaration

    Swift

    init(estMaitre: Bool, joueur: Joueur, position: (Int, Int))

    Parameters

    estMaitre

    true si le pion est un maître, sinon false
    joueur

    Le joueur auquel appartient ce pion
    position

    La position du pion sur le plateau

    Indique si le pion est un maître.

    Return Value

    true si le pion est un maitre, sinon false

    Indique à quel joueur appartient le pion.

  • enVie

    Indique si pion est toujours en vie. C'est à dire qu'il ne s'est pas encore fait manger par un autre pion.

    Declaration

    Swift

    var enVie: Bool { get }

    Return Value

    true si le pion est en vie, sinon false

  • position

    Indique la position du pion sur le plateau.

    Dans le cas d'utilisation du set, le tuple donné est de la forme (x, y) avec x appartenant à [0 ; 4] et y appartenant à [0 ; 4]. Si x ou y ne respectent pas ces conditions, alors on doit déclencher une erreur fatale.

    Lors de l'utilisation du get, on s'attend à ce que le retour soit égal à la position donnée précédemment lors de l'utilisation du set.

    Important

    La position est un tuple de la forme (x, y) avec x appartenant à [0 ; 4] et y appartenant à [0 ; 4]

    Declaration

    Swift

    var position: (Int, Int) { get set }

    Return Value

    un tuple d'entier, le premier élément du tuple est la position x et le second élement du tuple est y

  • tuer()

    Tue le pion. Si le pion n'est pas en vie (self.enVie == false), alors cette méthode ne fait rien. Si le pion est en vie (self.enVie == true), alors cette méthode fait le nécessaire pour que les appels suivant à self.enVie renvoient false.

    Important

    self.tuer() => self.enVie == false

    Declaration

    Swift

    mutating func tuer()
