Onitama

public protocol Onitama

Ce protocol représente le controlleur du jeu.

  • init()

    Crée un controlleur pour le jeu Onitama.

    Suite à init() on a :

    // L'accès à self.carteFlottante renvoie une erreur fatale
// L'état de self.plateau est le même que celui suivant l'initialisation du protocol Plateau
self.joueur1.estJoueur1() == true
self.joueur2.estJoueur1() == false

    Declaration

    Swift

    init()
  • joueur1

    Cette propriété représente le joueur 1.

    L’appel du get renvoie le joueur 1 c’est à dire le joueur tel que self.joueur1.estJoueur1() == true Le set n’est ici que pour satisfaire la mutabilité du joueur.

    Declaration

    Swift

    var joueur1: Joueur { get set }
  • joueur2

    Cette propriété représente le joueur 2.

    L’appel du get renvoie le joueur 2 c’est à dire le joueur tel que self.joueur2.estJoueur1() == false Le set n’est ici que pour satisfaire la mutabilité du joueur.

    Declaration

    Swift

    var joueur2: Joueur { get set }
  • carteFlottante

    Indique quelle carte est actuellement la carte flottante, c'est à dire la seule des cinq cartes du jeu qui ne soit pas en possession d'un des deux joueurs.

    Lors de l'appel de get on s'attend à recevoir la carte donnée précédemment via un appel de set. Si set n'a jamais été appelé et qu'on appelle get, alors on émet une erreur fatale.

    Declaration

    Swift

    var carteFlottante: Carte { get set }

    Return Value

    la carte flottante de la partie

  • tirer5Carte(parmi:)

    Tire 5 cartes au hasard parmi les cartes données.

    Si la liste de carte donnée en paramètre possède moins de 5 cartes, alors cette méthode émet une erreur fatale.

    Declaration

    Swift

    func tirer5Carte(parmi cartes: [Carte]) -> [Carte]

    Parameters

    cartes

    Une liste de cartes parmi lesquelles les 5 cartes doivent être choisies

    Return Value

    une liste de 5 cartes tirées au hasard parmi les cartes données

  • estTermine()

    Indique si la partie est terminée.

    La partie est terminée si le pion maître d'un joueur est mort ou si le pion maître d'un joueur est arrivé sur la case temple de l'autre joueur. La case temple du joueur1 se trouve en (2, 4) et la case temple du joueur2 se trouve en (2, 0).

    Declaration

    Swift

    func estTermine() -> Bool

    Return Value

    true si la partie est terminée, sinon false

  • plateau

    Cette propriété est le tableau sur lequel se déroule la partie. Cette propriété doit être initialisée lors de l'initilisation de ce type. Le set n’est ici que pour satisfaire la mutabilité du plateau.

    Declaration

    Swift

    var plateau: Plateau { get set }
